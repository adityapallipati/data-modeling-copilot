1. NAMING CONVENTIONS
-------------------
a) Logical Name Format:
   - Title Case (First Letter Each Word Capitalized)
   - Full Words (No Abbreviations)
   - Spaces Between Words
   - Maximum 50 Characters

b) Physical Name Format:
   - UPPER_SNAKE_CASE
   - Maximum 30 Characters
   - Only Underscores as Special Characters
   - Must End with Type Suffix

2. DATA TYPE STANDARDS
--------------------
a) Identifiers:
   - NUMBER(38,0) for Surrogate Keys
   - VARCHAR(50) for Natural Keys
   - Suffix: _ID, _NUM, _KEY

b) Dates and Timestamps:
   - DATE for Calendar Dates
   - TIMESTAMP_NTZ for Timestamps
   - Suffix: _DT, _TS

c) Numeric Measures:
   - NUMBER(10,8) for Probabilities/Scores
   - NUMBER(15,2) for Currency/Amounts
   - NUMBER(5,2) for Percentages
   - Suffix: _PCT, _AMT, _SCORE

d) Categorical:
   - VARCHAR(1) for Flags
   - VARCHAR(20) for Codes
   - VARCHAR(100) for Descriptions
   - Suffix: _FLAG, _CD, _DESC

3. COMMON ABBREVIATIONS
---------------------
cust -> CUSTOMER
dt -> DATE
ts -> TIMESTAMP
num -> NUMBER
amt -> AMOUNT
feat -> FEATURE
imp -> IMPORTANCE
pred -> PREDICTION
grp -> GROUP
bkt -> BUCKET
prob -> PROBABILITY

4. DOMAIN SPECIFIC
----------------
a) ML Model Outputs:
   - Probabilities: NUMBER(10,8), suffix _PROBABILITY
   - Feature Importance: NUMBER(10,8), suffix _IMPORTANCE
   - Predictions: Match Target Variable Type
   - Confidence: NUMBER(10,8), suffix _CONFIDENCE
   - Model Metadata: VARCHAR(50), suffix _CD

5. NULLABILITY
------------
- Primary Keys: NOT NULL
- Foreign Keys: NOT NULL
- Timestamps: NOT NULL for Audit Fields
- Scores/Probabilities: NOT NULL
- Optional Attributes: NULL Allowed
